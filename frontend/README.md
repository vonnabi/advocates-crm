# Frontend

Текущий интерфейс CRM. В рабочем локальном режиме он открывается через Django backend:

```text
http://127.0.0.1:8001/
```

Отдельно запускать frontend не нужно: Django на `8001` отдает и HTML/CSS/JS, и API.

## Файлы

- `index.html` — структура страницы.
- `styles.css` — визуальный стиль.
- `app.js` — демо-логика и данные.

## Как редактировать

Если нужно поменять текст, названия разделов или структуру блоков — чаще всего это `app.js`.

Если нужно поменять цвета, отступы, размеры, карточки, меню — это `styles.css`.

Если нужно добавить новый общий HTML-блок или подключить новый файл — это `index.html`.

## Стиль модалок

Для новых и постепенно обновляемых всплывающих окон используем общий набор классов:
`crm-modal-form`, `crm-modal-head`, `crm-modal-title`, `crm-modal-title-icon`, `crm-modal-close`,
`crm-modal-status`, `crm-modal-grid`, `crm-modal-field`, `crm-modal-actions`.

Первый эталон этого стиля — окно настроек интеграции в разделе `Налаштування`.
Дальше остальные модалки переводим на эту же структуру, чтобы заголовок, кнопка закрытия,
статусный блок, поля и кнопки были выровнены одинаково.

## Важно

Основные данные в рабочем режиме приходят из Django API и базы данных. JSON-файлы в `frontend/data/` остаются запасным демо-режимом для чистого frontend-прототипа.
