# Demo data

These files contain the static demo data used by the frontend prototype.

- `clients.json` - client records and communication history.
- `cases.json` - cases, documents, tasks, history, finance fields.
- `events.json` - calendar events.
- `mailing.json` - mailing text, filters, test contacts, automation rules.
- `settings.json` - bureau profile, integrations, notification defaults.

Keeping data here makes `app.js` easier to read and prepares the prototype for a future backend/API migration.
