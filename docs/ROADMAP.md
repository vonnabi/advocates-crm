# План разработки

Актуальный срез готовности находится в `docs/READINESS_AUDIT.md`.

На 27.05.2026 проект уже вышел за рамки первого кликабельного прототипа: Django backend, API, Render, документы, финансы, календарь, задачи, рассылки и настройки работают как единая CRM-структура. Ниже оставлена история этапов и обновленный порядок следующих работ.

## Этап 1. Быстрый прототип

Статус: начато.

Цель:

- собрать интерфейс по скринам;
- показать основные разделы;
- согласовать структуру CRM;
- не тратить время на сервер до согласования логики.

Готово:

- базовый интерфейс;
- демо-данные;
- разделы CRM;
- добавление клиента;
- добавление события;
- поиск клиентов;
- предпросмотр рассылки.

## Этап 2. Django backend

Цель:

- подключить настоящую базу данных;
- сохранить клиентов, дела, события и задачи;
- добавить пользователей и роли.

Первые Django apps:

- `accounts` — пользователи и роли;
- `clients` — база клиентов;
- `cases` — дела;
- `calendar_app` — события и напоминания;
- `tasks` — планер;
- `finance` — счета, платежи, долги;
- `communications` — рассылки и история сообщений.

## Этап 3. Интеграции

Подключить:

- Telegram Bot API;
- SMS-провайдера;
- Email;
- фоновые задачи для отправок и напоминаний.

## Этап 4. Документы и AI

Добавить:

- загрузку PDF/DOCX/XLSX;
- карточку документов по делу;
- AI-помощников по отраслям права;
- AI-помощников по конкретному делу;
- генерацию Word-документов.

## Этап 5. Продакшн

Нужно будет сделать:

- деплой на сервер;
- домен;
- HTTPS;
- резервные копии;
- журнал действий;
- защиту персональных данных;
- мониторинг ошибок.

## Текущий рабочий этап

Статус: пилотная подготовка.

Главная цель:

- убрать остатки макетной логики;
- закрепить золотые сценарии тестами;
- довести финансы, отчеты, рассылки и интеграции до состояния, которое можно показывать на реальных данных.

Уже закрыто:

- Django backend и API;
- локальный запуск на `127.0.0.1:8001`;
- Render Blueprint;
- GitHub Pages для статического просмотра;
- клиенты, дела, задачи, календарь, документы и финансы;
- demo-toggle без удаления ручных данных;
- ONLYOFFICE-логика для документов;
- счета и акты как физические документы дела;
- smoke/golden-тесты для ключевых сценариев.

Следующий порядок:

1. Финансы: отчеты, зарплата, чистая аналитика без демо-графиков.
2. Рассылка: проверить шаблоны, прогноз, статусы и подготовить реальные каналы.
3. Роли: прогнать администратора, адвоката, помощника и бухгалтера.
4. Интеграции: Telegram, Email, SMS, КЕП, ONLYOFFICE в боевом режиме.
5. AI/OSINT: определить реальные источники и сценарии работы.
