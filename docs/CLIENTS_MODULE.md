# Модуль: База клиентов

## Назначение

Раздел хранит клиентов юридического бюро, их обращения, контакты, привязанные дела, события, историю коммуникаций и согласия на рассылки.

## Поля клиента для будущей Django-модели

Основные поля:

- `id`
- `full_name`
- `client_type`
- `phone`
- `email`
- `telegram_username`
- `telegram_connected`
- `request_summary`
- `status`
- `source`
- `responsible_user`
- `marketing_consent`
- `risk_level`
- `next_action`
- `notes`
- `created_at`
- `last_contact_at`

Связи:

- клиент имеет много дел;
- клиент имеет много событий календаря;
- клиент имеет много коммуникаций;
- клиент имеет финансовые операции через дела;
- клиент может участвовать в рассылках.

## Статусы клиента

- `Новий`
- `Активний`
- `Постійний клієнт`
- `Не турбувати`

## Уровни риска

- `Низький`
- `Середній`
- `Високий`

## Коммуникации

Каждая запись истории коммуникации должна хранить:

- дату;
- канал;
- тему / описание;
- статус;
- автора;
- привязку к клиенту;
- опционально привязку к делу.

Каналы:

- звонок;
- Telegram;
- SMS;
- Email;
- CRM-заметка;
- встреча.

## Что уже реализовано в прототипе

- таблица клиентов;
- фильтр по статусу;
- фильтр по источнику;
- поиск;
- карточка клиента справа;
- контакты;
- Telegram username;
- согласие на рассылку;
- риск;
- следующая задача;
- дела клиента;
- события клиента;
- история коммуникаций;
- добавление нового клиента.
- редактирование клиента.
