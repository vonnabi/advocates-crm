# Карта проекта

Этот файл нужен, чтобы быстро понимать, где что искать в проекте.

## `frontend/`

Текущий рабочий прототип интерфейса.

- `index.html` — основная HTML-страница.
- `styles.css` — внешний вид: меню, карточки, таблицы, календарь, формы.
- `app.js` — демо-логика: данные, переключение разделов, добавление клиентов и событий.

Пока это самостоятельный прототип без сервера и базы данных. Позже этот интерфейс можно перенести на React/Next.js или связать с Django templates.

## `backend/`

Будущий Django backend.

Здесь будут:

- Django project;
- Django apps по модулям CRM;
- модели базы данных;
- API;
- авторизация;
- права доступа;
- фоновые задачи;
- интеграции.

## `data/`

Демо-данные и seed-файлы.

Сюда стоит складывать:

- примеры клиентов;
- примеры дел;
- статусы;
- типы событий;
- шаблоны рассылок;
- справочники.

## `materials/`

Исходные материалы проекта.

Здесь лежат:

- техническое задание;
- исходный WhatsApp-чат;
- RTF-файлы по отдельным разделам;
- все картинки из выгрузки;
- выбранные референсы по модулям CRM.

Главные подпапки:

- `technical-task/` — ТЗ и текстовые исходники.
- `screenshots/` — скриншоты и картинки.
- `screenshots/all-originals/` — все оригинальные изображения без отбора.

## `docs/`

Вся документация проекта.

- карта проекта;
- описание модулей;
- план разработки;
- архитектура;
- заметки по ТЗ.

## `design/`

Материалы по дизайну.

Сюда можно сложить:

- скрины из WhatsApp;
- референсы интерфейса;
- будущие макеты;
- заметки по цветам, таблицам, карточкам.

## `integrations/`

Описание внешних сервисов.

Тут будут инструкции по:

- Telegram Bot API;
- SMS-провайдеру;
- Email-сервису;
- OpenAI/AI API.

## `scripts/`

Вспомогательные команды.

Например:

- запуск локального сервера;
- импорт демо-данных;
- создание тестового администратора;
- резервное копирование.
