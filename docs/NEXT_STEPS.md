# Ближайшие шаги

## Что у нас сейчас есть

Проект уже можно смотреть в браузере:

- есть рабочий frontend-прототип;
- есть основные разделы CRM;
- есть структура папок под нормальную разработку;
- есть место под Django backend;
- есть исходное ТЗ и первые скрины.

## С чего начинаем дальше

### Шаг 1. Довести frontend-прототип до понятной версии

Нужно пройтись по каждому разделу и решить:

- какие поля точно нужны;
- какие кнопки должны работать;
- какие таблицы должны быть;
- какие карточки открываются справа;
- какие статусы и фильтры нужны.

Первый модуль для доведения: `База клиентов`.

Почему с него:

- клиенты нужны почти всем другим разделам;
- дела привязываются к клиентам;
- события календаря привязываются к клиентам;
- рассылки идут по клиентам;
- финансы считаются по клиентам.

### Шаг 2. Разделить данные frontend на отдельные файлы

Демо-данные больше не должны жить внутри большого `frontend/app.js`.

Нужно вынести их в:

- [x] `frontend/data/clients.json`;
- [x] `frontend/data/cases.json`;
- [x] `frontend/data/events.json`;
- [x] `frontend/data/mailing.json`;
- [x] `frontend/data/settings.json`;
- [ ] отдельно разнести производные задачи/финансы, если их станет больше.

Это поможет потом легче перенести данные в Django.

### Шаг 3. Сделать проект Django

После согласования frontend-логики создаем настоящий backend:

- `manage.py`;
- настройки Django;
- базу SQLite на старте;
- приложения `clients`, `cases`, `calendar_app`, `tasks`, `finance`.

### Шаг 4. Подключить frontend к backend

Frontend перестает хранить данные внутри `app.js` и начинает получать их из Django.

Первые API:

- список клиентов;
- создание клиента;
- список дел;
- список событий;
- создание события.
