# UI-паттерны CRM

## Статусы в таблицах

Решение, которое зафиксировано на экранах `Документы`, `Задачи` и `Справи`: в плотных таблицах статус лучше показывать компактной цветной иконкой, а не широким текстовым бейджем.

Почему так:

- экономит место в строке;
- таблица становится ровнее по вертикали;
- статус быстрее считывается глазами;
- длинные подписи не ломают сетку;
- текст статуса остается доступен через мягкую подсказку при наведении.

## Правило отображения

В таблицах:

- показываем только круглую иконку статуса;
- используем отдельный цвет для каждого состояния;
- добавляем короткую подсказку при наведении;
- не используем нативный `title`, чтобы не появлялись лишние системные подсказки;
- курсор оставляем обычным, без знака вопроса.

В правой карточке:

- оставляем выпадающую кнопку со статусом;
- показываем и иконку, и текст;
- активный статус подсвечиваем цветом;
- стрелка раскрытия должна быть аккуратной и в одну линию с текстом.

## Цвета и смысл

Для документов текущая логика такая:

- `Чернетка` — фиолетовый карандаш;
- `Не подано` — янтарные часы;
- `В роботі` — синее обновление;
- `Подано` — голубой документ со стрелкой вверх, то есть документ передан/подан;
- `Відповідь очікується` — желтый колокольчик;
- `Потрібно перевірити` — красная лупа;
- `Отримано` — бирюзовый документ;
- `Готово` — насыщенная зеленая галочка как финальное действие.

Для задач текущая логика такая:

- `Нова` — фиолетовая метка;
- `Заплановано` — голубой календарь;
- `Очікує` — желтые часы;
- `В роботі` — синее обновление;
- `На перевірці` — красная лупа;
- `Перенесено` — янтарный календарь;
- `Просрочено` — красное предупреждение-треугольник;
- `Терміново` — красный колокольчик;
- `Скасовано` — красная корзина;
- `Не терміново` — зеленые часы;
- `Виконано` — насыщенная зеленая галочка как финальное действие.

Для справ текущая логика такая:

- `В роботі` — синее обновление;
- `Очікує відповідь` — желтые часы;
- `Терміново` — красный колокольчик;
- высокий приоритет — красный колокольчик;
- средний приоритет — желтый колокольчик;
- низкий приоритет — зеленый колокольчик;
- плановый приоритет — фиолетовый колокольчик.

## Массовые действия

Паттерн из задач и документов:

- чекбокс в заголовке выбирает видимые строки;
- рядом с заголовком появляется компактная панель действий;
- панель не должна доходить до следующего столбца;
- действия лучше делать иконками в цвете;
- текст действия показываем только в подсказке;
- последняя кнопка — закрыть/снять выбор;
- удаление ставим перед закрытием, чтобы логически отделить опасное действие от сброса выбора.

## Справи: клиент первым

Для меню `Справи` основной ориентир в списке — клиент, а номер дела вторичный, но таблица остается плоской и компактной.

Правило:

- первый столбец — клиент;
- второй столбец — номер справы;
- дальше идут стандартные рабочие поля: название, этап, дедлайн, приоритет, статус, финансы;
- поиск в первую очередь должен уверенно находить клиента, а номер дела остается дополнительным способом поиска;
- не добавляем отдельную строку-группу клиента, если она перегружает таблицу.

Этот подход сейчас лучше для плотной юридической таблицы: клиент виден сразу, но строки остаются привычными и быстрыми для сканирования.

## Где применять дальше

Этот паттерн стоит постепенно перенести на:

- клиентов;
- календарные события;
- финансы;
- AI/OSINT списки;
- любые таблицы, где есть массовый выбор и статусы.

Главный принцип: в таблице меньше текста, больше стабильных иконок; в карточке справа можно показывать больше расшифровки.
